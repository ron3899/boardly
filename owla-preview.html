🚀 New Feature

Mock Auth Mode

Database-free authentication for sandbox & development environments

✨ Overview

Mock Auth Mode enables the Boardly application to run completely standalone without requiring a PostgreSQL database connection. Perfect for:

  • ✓ Local development without database setup
  • ✓ CI/CD pipeline testing
  • ✓ Quick demos and prototyping
  • ✓ Onboarding new developers

🔧 How It Works

When MOCK_AUTH=true is set, the backend uses an in-memory user store instead of Prisma/PostgreSQL. All auth endpoints work identically.

🎯 Key Features

  • ✓ Pre-seeded demo user
  • ✓ Full register/login/logout flow
  • ✓ JWT token issuance & validation
  • ✓ Bcrypt password hashing
  • ✓ Zero database dependencies

📋 Implementation Flow

1
Environment Configuration
Set MOCK_AUTH=true in apps/api/.env
↓
2
Service Initialization
MockAuthService loads with pre-seeded demo user
↓
3
User Login
POST /auth/login validates credentials against in-memory store
↓
4
JWT Token Issued
Server sets httpOnly cookie with JWT
↓
5
Authenticated Access
GET /auth/me returns user data from memory

🔐 Security Features

  • ✓ Passwords hashed with bcrypt (12 rounds)
  • ✓ JWT tokens with secure httpOnly cookies
  • ✓ Same validation logic as production auth
  • ✓ Proper error handling (401, 404, 409)

⚙️ Backend Configuration

Environment Setup

MOCK_AUTH=true
JWT_SECRET=dev-secret-key-boardly
PORT=3001
CORS_ORIGIN=http://localhost:3000

MockAuthService

export class MockAuthService { private users: Map<string, MockUser> async login(input: LoginInput) { const user = this.users.get(userId) const valid = await bcrypt.compare( input.password, user.passwordHash ) // ... validation & return user } }
