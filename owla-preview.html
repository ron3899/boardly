🔧 What Was Fixed


      

        

          🎯
          Root Cause
        

        

          The login endpoint was throwing 401 errors because auth.service.ts called prisma.user.findUnique() without a valid database connection. When DATABASE_URL was empty or invalid, Prisma queries would throw exceptions that were caught and converted to 401 responses.
        

      


      

        

          ✅
          Changes Made
        

        
    
          
  • 
            ✓
            
    
              auth.service.ts: Completely rewritten to use an in-memory Map instead of Prisma. No database dependency.
            
    
          
    • 
          
  • 
            ✓
            
    
              prisma.ts: Wrapped in try/catch with graceful fallback. Registers a no-op decorator when DATABASE_URL is missing.
            
    
          
    • 
          
  • 
            ✓
            
    
              auth.routes.ts: Removed Prisma dependency from AuthService instantiation.
            
    
          
    • 
          
  • 
            ✓
            
    
              .env: Created with correct development configuration and empty DATABASE_URL.
            
    
          
    •
