apps/web/.env

NEXT_PUBLIC_API_URL=http://localhost:3001

This environment variable tells the Next.js web app where to find the API server. For local development, it points to localhost:3001.

apps/api/.env

DATABASE_URL="postgresql://user:password@localhost:5432/boardly"
JWT_SECRET="dev-secret-change-me-in-production"
PORT=3001
CORS_ORIGIN="http://localhost:3000"

The API server configuration includes the allowed CORS origin. This must match the web app's URL for CORS to work with credentials.

apps/api/src/plugins/cors.ts

app.register(fastifyCors, {
  origin: (process.env.CORS_ORIGIN || 'http://localhost:3000').split(','),
  credentials: true,
})

The CORS plugin is configured to allow specific origins (not wildcard *) and enable credentials for cookie-based authentication. Multiple origins can be specified using comma-separated values.

⚠️ Why Wildcard CORS Doesn't Work with Credentials

❌ Incorrect Configuration

origin: '*',
credentials: true

Browsers block this combination for security. You cannot use wildcard origin when credentials are enabled.

✅ Correct Configuration

origin: ['http://localhost:3000'],
credentials: true

Specific origins are required with credentials. This ensures secure cookie-based authentication.

🚀 Development Setup

Quick Start

# 1. Environment files are already created
# (apps/web/.env and apps/api/.env)

# 2. Install dependencies
pnpm install

# 3. Setup database (update DATABASE_URL first)
cd apps/api
pnpm db:push
pnpm db:seed

# 4. Start development servers
cd ../..
pnpm dev

This will start both the web app on http://localhost:3000 and the API server on http://localhost:3001 with proper CORS configuration.

📋 Production Deployment

Environment Variables

For production deployment, update the environment variables to match your deployed URLs:

Web App (Vercel)

NEXT_PUBLIC_API_URL=https://your-api.onrender.com

API Server (Render)

CORS_ORIGIN=https://your-app.vercel.app

💡 Tip: You can specify multiple origins separated by commas for staging and production environments.
